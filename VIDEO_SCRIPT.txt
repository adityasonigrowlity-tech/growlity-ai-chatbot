--- 2.3: THE AI BRAIN — aiService.js ---

[SCREEN: Open Backend/services/aiService.js]

"This is the core AI brain of the chatbot. This file handles everything
related to AI response generation.

  FILE: Backend/services/aiService.js
  DEPENDENCIES: @langchain/openai, @langchain/groq, @langchain/core

Here's how it works step-by-step:

  STEP 1 — SYSTEM PROMPT:
  At the top, we define a detailed System Prompt that tells the AI to act
  as 'Growlity AI Chatbot' — a professional ESG consultant. It includes
  specific guidelines like response length, tone, and when to share
  links to Growlity's pages.

  STEP 2 — RAG (Retrieval Augmented Generation):
  The function 'getRelevantContext()' is the RAG engine.
  When a user asks a question:
    1. The question is converted into a vector using OpenAI Embeddings
       (model: text-embedding-3-small, hosted via OpenRouter).
    2. This vector is used to run a $vectorSearch query against MongoDB Atlas.
       The search index is named 'AiChatbot' and it returns the top 5
       most relevant knowledge chunks based on cosine similarity.
    3. These chunks become the 'context' that is injected into the prompt.

  STEP 3 — LLM RESPONSE with FALLBACK:
  The function 'generateResponse()' sends the messages to the AI:
    • Primary: Groq API running Llama 3.3 70b (ultra-fast, ~0.5s response)
    • Fallback: OpenRouter running GPT-oss-120b (if Groq fails)
  It also includes full conversation history for multi-turn context.

  If both models fail, a graceful error message is returned."


--- 2.4: WEB SCRAPING — scraper.js ---

[SCREEN: Open Backend/services/scraper.js]

"This file handles web scraping — it's how we feed knowledge into the
chatbot.

  FILE: Backend/services/scraper.js
  DEPENDENCIES: axios, cheerio

  'scrapeGrowlity()' takes a URL, fetches the page content (optionally
  through Scrape.do API for anti-bot protection), and then uses Cheerio
  to parse the HTML. It extracts clean text from headings, paragraphs,
  lists, and tables while removing headers, footers, and navigation.

  'chunkText()' breaks the scraped text into overlapping chunks of
  800 characters with 150-character overlap. This ensures no context
  is lost at chunk boundaries when stored in the database."


--- 2.5: TEXT-TO-SPEECH — ttsService.js ---

[SCREEN: Open Backend/services/ttsService.js]

"This file powers the voice feature.

  FILE: Backend/services/ttsService.js
  DEPENDENCY: axios (calls ElevenLabs API)

  It sends the AI-generated text to ElevenLabs' Text-to-Speech API
  using the 'eleven_turbo_v2_5' model. The audio is streamed back
  as an MP3 and played directly in the browser."


--- 2.6: KNOWLEDGE INGESTION — ingestController.js ---

[SCREEN: Open Backend/controllers/ingestController.js]

"This controller handles how new knowledge gets added to the chatbot.

  FILE: Backend/controllers/ingestController.js
  DEPENDENCIES: scraper.js, Knowledge model, @langchain/openai

  It exposes two main endpoints:
    • 'ingestUrl'  — Scrapes a URL, chunks the text, generates embeddings
                     for each chunk, and stores them in MongoDB
    • 'ingestText' — Same flow but for manually pasted text

  There are also endpoints to list all knowledge entries and to delete
  specific entries or entire sources at once.

  All ingest routes are protected by JWT authentication via the auth
  middleware — only logged-in admins can modify the knowledge base."


--- 2.7: DATABASE MODEL — Knowledge.js ---

[SCREEN: Open Backend/models/Knowledge.js]

"The knowledge is stored in MongoDB using this Mongoose model.

  FILE: Backend/models/Knowledge.js
  DEPENDENCY: mongoose

  Each document has three key fields:
    • 'content'   — The actual text chunk
    • 'metadata'  — Source URL/title, chunk index, and type
    • 'embedding' — A numeric vector array (1536 dimensions)
                    used for MongoDB Atlas Vector Search"


--- 2.8: SERVER & ROUTING — server.js ---

[SCREEN: Open Backend/server.js]

"The main server file ties everything together.

  FILE: Backend/server.js
  KEY MIDDLEWARE: CORS, rate-limiting, JSON parsing, JWT auth

  API Routes:
    • /api/chat    — Handles user messages (rate-limited)
    • /api/tts     — Text-to-Speech requests
    • /api/ingest  — Knowledge management (JWT-protected)
    • /api/auth    — Admin login/authentication
    • /health      — Health check endpoint

  The server includes clean shutdown logic that properly closes
  both the HTTP server and MongoDB connection."


--- 2.9: FRONTEND — ChatWidget.jsx ---

[SCREEN: Open Frontend/src/components/ChatWidget.jsx]

"On the frontend, the entire chatbot is a single React component
called ChatWidget.

  FILE: Frontend/src/components/ChatWidget.jsx
  DEPENDENCIES: react, axios, lucide-react, react-markdown, remark-gfm

  It handles:
    • Chat UI with animated message bubbles
    • Sending questions to the backend via Axios
    • Rendering AI responses as rich Markdown (with GFM support)
    • Text-to-Speech playback with audio controls
    • Message editing and re-sending
    • Auto-scroll with a 'scroll to bottom' button
    • Responsive design — full-width on mobile

  The widget is built and exported as a UMD bundle (grow-ai-widget.js)
  using Vite, so it can be embedded on any website — including WordPress
